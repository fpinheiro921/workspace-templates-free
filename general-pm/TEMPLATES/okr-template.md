## OKR Quality Checklist
Before finalizing, confirm:
- [ ] Each Objective is inspiring and directional (not a task)
- [ ] Each KR is measurable with a specific number
- [ ] KRs are outcomes, not outputs ("reduce churn by 10%" not "ship retention feature")
- [ ] We know exactly how we'll measure each KR
- [ ] Someone owns each KR
- [ ] We've challenged: "Could we hit these KRs while building the wrong things?"
