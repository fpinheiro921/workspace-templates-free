# Subject Lines

Every newsletter has 5 subject-line variants generated. User picks one.

Full framework in `_PATTERNS/hook-library-core.md` Part A (6-piece headline) + Part D (compression rules).

## The 8 patterns

Rotate these. Don't use the same pattern 3 sends in a row — the list starts pattern-matching.

### Pattern 1 — Number + Noun + Topic
`7 tips for your first client`

### Pattern 2 — How to + Outcome
`how to niche without overthinking`

### Pattern 3 — Declarative claim
`your niche is your seed`

### Pattern 4 — Controversial "stop doing X"
`stop writing for everyone`

### Pattern 5 — Short question
`are you still a generalist?`

### Pattern 6 — Curiosity gap
`the 20% rule I broke for a year`

### Pattern 7 — Confession / vulnerability
`I stalled at $3k for 18 months`

### Pattern 8 — Insider / behind-the-scenes
`what I learned from 50 sales calls`

## Writing rules

- **Length:** 30-50 characters / 4-8 words
- **Case:** sentence case — capitalize first word only
- **Starts-with-number exception:** if starting with a number, the next word stays lowercase
- **Preview text:** separately draft a 70-120-char preview that extends (not repeats) the subject
- **No emojis** (unless your voice guide allows)

## Testing

When platform allows (Substack Plus, Beehiiv, Kit):
- A/B test one variable at a time (pattern / word count / punctuation)
- Need 3-5 wins in a pattern to call it a trend (one-off wins don't mean much)
- Keep running log in `analytics.md`

## Red flags

- Any subject line with ALL CAPS WORDS → probably spam-flagged
- Starts with "RE:" or "FWD:" (unless it really is a reply) → manipulation tactic, kills trust
- Clickbait that doesn't deliver in the body → open rate up, unsubscribe rate up worse
- Over-4-emoji subject lines → scream "marketing email"
