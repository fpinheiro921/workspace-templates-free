# AUTOMATION/

Course Module 3. The operational infrastructure that turns "I write a newsletter" into "I run a newsletter".

## Files

| File | Purpose |
|---|---|
| `workflow.md` | The platform-specific automation stack + tool decisions |
| `welcome-sequence.md` | 10-email onboarding for new subscribers |
| `platform-templates.md` | Platform-native templates (Substack / Beehiiv / Ghost / Kit) — which to use + customizations |
| `checklists.md` | Pre-send, post-send, weekly review |

## The principle

Automate the repetitive, preserve the judgment-heavy.

**Automate:**
- Welcome email sequence
- "What happens after someone hits Publish"
- Recurring analytics pulls
- Sponsored-content delivery artifacts
- Referral tracking

**Don't automate:**
- The actual writing
- Sending a specific reader a thoughtful reply
- Deciding what goes in next week's issue
- Pricing changes
- Sponsor approvals

## Cascade behavior

- When user says "set up X automation", route to the specific file
- Never make live changes to platform automations — Cascade drafts the email copy, user configures in-platform
- Pre-send checklist enforced before every issue goes out
