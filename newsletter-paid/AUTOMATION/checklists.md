# Checklists

Pre-send, post-send, weekly review. Print-and-use or reference from Cascade.

## Pre-send (every newsletter, before hitting Publish)

- [ ] Two-Pass Review complete (Craft + Voice from `_PATTERNS/two-pass-review.md`)
- [ ] 5 subject-line variants generated, 1 selected
- [ ] Preview text (70-120 chars) written and doesn't duplicate subject
- [ ] Hook lands in first 1-2 sentences (one of 6 types)
- [ ] Word count within target range
- [ ] Banned-word audit clean (scan against `CONTEXT/voice-guide.md`)
- [ ] All links working (click every one; use UTM for tracked links)
- [ ] Image alt-text where applicable
- [ ] Sender name correct (your name, not "newsletter")
- [ ] From address correct
- [ ] Signature block includes reply-invitation
- [ ] Sponsor copy (if any) approved by sponsor + matches their spec
- [ ] Affiliate disclosures in place (if any)
- [ ] Unsubscribe link works
- [ ] If paid-tier-gated content: actually gated

**Rule:** if ANY unchecked, do not hit Publish. No "I'll fix it after."

## Post-send (within 24h)

- [ ] Screenshot of sent email saved (for sponsor records if applicable)
- [ ] Derivative content drafted (via `PROMPTS/generators/repurposing-prompt.md`)
- [ ] Derivatives scheduled on secondary platforms (Twitter, LinkedIn)

## Post-send (+48h)

- [ ] Analytics entry started in `PUBLISHING/analytics.md`
  - Opens, open rate, clicks, replies, subs added, unsubs
  - Qualitative notes from replies

## Post-send (+7 days)

- [ ] Analytics entry completed
  - Cumulative metrics
  - Baseline comparison
  - Lessons extracted
- [ ] If sponsored: metrics report sent to sponsor
- [ ] Draft moved to `PUBLISHED/`

## Weekly review (Friday 30 min)

- [ ] Check this week's sent issue metrics
- [ ] Update `MONETIZATION/` monthly totals (cumulative this month)
- [ ] Review next week's calendar entry — is the draft started?
- [ ] Scan inbound DMs / replies — any need a dedicated response?
- [ ] Growth metrics: sub-adds this week, attributable to which lever?
- [ ] `AUDIENCE/engagement-targets.md` review (newsletter-paid doesn't have AUDIENCE/ — cross-reference `creator-studio/` if active)

## Monthly review (last Friday of month, 60 min)

- [ ] Aggregate month's metrics across all issues
- [ ] Top-performing issue + why
- [ ] Bottom-performing issue + why
- [ ] Any red flags in unsubscribe or open-rate trends?
- [ ] `MONETIZATION/` — monthly revenue per lane
- [ ] Update `CONTEXT/newsletter-brief.md` subscriber count + revenue current state
- [ ] Any decisions for `CONTEXT/decisions-log.md`?

## Quarterly review (end of week 12/24/36/48, 2 hours)

- [ ] Full `newsletter-brief.md` review — still the right niche?
- [ ] `POSITIONING/competitive-analysis.md` refresh — new competitors?
- [ ] `CONTEXT/monetization-thesis.md` review — on track?
- [ ] Next quarter's `90-day-calendar.md` built from scratch (not copy-paste)
- [ ] Retire 1 automation / tool / lane that isn't earning its keep
- [ ] Goals for next quarter — subscribers, revenue, 1-3 strategic bets

## Cascade behavior

- Every Friday morning: Cascade surfaces "Weekly review time — ready?"
- Last Friday of month: "Monthly review time"
- Week 11/23/35/47: "Quarterly review is coming up — want to set aside 2 hours?"
- Never auto-skips a review even if user is busy; flags it as technical debt.
