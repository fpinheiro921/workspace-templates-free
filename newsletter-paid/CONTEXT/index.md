# CONTEXT/

Who this newsletter is for, why it exists, and the big strategic decisions that shape everything downstream.

## Read order (on project start)

1. `newsletter-brief.md` — the one-page summary: niche + platform + stage + revenue target
2. `voice-guide.md` — how you sound (tone, length, mechanics)
3. `monetization-thesis.md` — which of the 5 lanes at which stage
4. `decisions-log.md` — reasoning trail for strategic pivots

## Files

| File | Purpose |
|---|---|
| `newsletter-brief.md` | The strategic one-pager — fill this FIRST |
| `voice-guide.md` | Voice rules, banned phrases, signature moves |
| `monetization-thesis.md` | Revenue plan: lanes × stages × targets |
| `decisions-log.md` | Dated log of strategic decisions + reasoning |

## Rule

Everything in CONTEXT is **user-written**, not Cascade-generated. Cascade can draft suggestions but the final wording here is always the user's. These files encode strategic intent — Cascade references them, she doesn't author them.

## Cascade behavior

- At session start, read all 4 files
- If any file has placeholders ("TBD", "<fill in>"), flag at session start and offer to help fill
- Never write strategic decisions into `decisions-log.md` without explicit user approval
